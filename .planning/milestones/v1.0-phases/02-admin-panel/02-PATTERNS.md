### `src/db/schema/translation-flags.ts` (NEW — D-05)

**Analog:** `src/db/schema/spec-values.ts:55-68` (`productSpecValueTranslations` sibling table on a composite PK)

**Pattern (one table per translation family — translation_id is composite or surrogate):**

Because `product_translations` PK is `(product_id, locale)` (a composite), the cleanest shape is to repeat the composite as the FK:
```typescript
import { pgTable, uuid, text, boolean, primaryKey } from 'drizzle-orm/pg-core';
import { productTranslations } from './products';

export const productTranslationFieldFlags = pgTable(
  'product_translation_field_flags',
  {
    productId: uuid('product_id').notNull(),
    locale: text().notNull(),
    fieldName: text('field_name').notNull(),
    machineTranslated: boolean('machine_translated').notNull().default(false),
  },
  (t) => [
    primaryKey({ columns: [t.productId, t.locale, t.fieldName] }),
    // Compound FK to product_translations(product_id, locale)
    foreignKey({
      columns: [t.productId, t.locale],
      foreignColumns: [productTranslations.productId, productTranslations.locale],
    }).onDelete('cascade'),
  ],
);
// Repeat for category_translation_field_flags, manufacturer_*, spec_field_*,
// product_spec_value_translation_field_flags, etc. as needed.
```

**Planner note:** Open Q §2 in RESEARCH.md recommends this sibling-table form over per-table boolean columns. Phase 2 ships flags only for `productTranslations` and `productSpecValueTranslations` (the two surfaces the editor exposes); `category`/`manufacturer`/`spec_field`/`spec_field_group` translation field flags can be deferred to v2.

### `src/db/schema/views/product-translation-completeness.ts` (NEW — D-04, ADMIN-10)

**Analog:** none in repo — greenfield. RESEARCH.md Pattern 6 is the canonical source.

**Import the new view to barrel:** `src/db/schema/index.ts` add `export * from './views/product-translation-completeness';`.

**Drizzle pgView pattern** (from RESEARCH.md §Pattern 6, lines 660-681):
```typescript
import { pgView, text, uuid, integer } from 'drizzle-orm/pg-core';
import { sql } from 'drizzle-orm';

export const productTranslationCompleteness = pgView(
  'product_translation_completeness',
  {
    productId: uuid('product_id').notNull(),
    locale: text('locale').notNull(),
    percent: integer('percent').notNull(),
  },
).as(sql`
  WITH base AS (
    SELECT pt.product_id, pt.locale,
           (CASE WHEN pt.name        IS NOT NULL AND length(pt.name)        > 0 THEN 1 ELSE 0 END +
            CASE WHEN pt.short_desc  IS NOT NULL AND length(pt.short_desc)  > 0 THEN 1 ELSE 0 END +
            CASE WHEN pt.long_desc   IS NOT NULL AND length(pt.long_desc)   > 0 THEN 1 ELSE 0 END +
            CASE WHEN pt.slug        IS NOT NULL AND length(pt.slug)        > 0 THEN 1 ELSE 0 END) AS filled,
           4 AS total
      FROM product_translations pt
  )
  SELECT product_id, locale, ROUND(filled::numeric / NULLIF(total,0) * 100)::int AS percent
    FROM base
`);
```

**Wrapper shape** (RESEARCH.md §Code Examples):
```typescript
export function withAdminAction<I, O>(
  schema: z.ZodSchema<I>,
  handler: (input: I, ctx: { actorEmail: string; ip: string; userAgent: string }) => Promise<O>,
) {
  return async (raw: unknown): Promise<{ ok: true; data: O } | { ok: false; error: string }> => {
    try {
      const session = await requireAdmin();
      const input = schema.parse(raw);
      const h = await headers();
      const data = await handler(input, {
        actorEmail: session.user!.email!,
        ip: h.get('x-forwarded-for') ?? 'unknown',
        userAgent: h.get('user-agent') ?? 'unknown',
      });
      return { ok: true, data };
    } catch (err) {
      console.error('admin-action', err);
      return { ok: false, error: err instanceof z.ZodError ? 'validation' : 'unauthorized' };
    }
  };
}
```

### `src/lib/audit.ts` (NEW — D-16)

**Analog (insert shape):** `src/lib/bootstrap.ts:43-46` (`db.insert(adminUsers).values(...)`)
**Analog (table reference):** `src/db/schema/admin.ts:24-35` (`auditLog` columns)

```typescript
import { auditLog } from '@/db/schema';
import type { dbTx } from '@/db/client-ws';
type Tx = Parameters<Parameters<typeof dbTx.transaction>[0]>[0];

export type AuditAction =
  | 'create' | 'update' | 'delete'
  | 'publish' | 'unpublish'
  | 'invite' | 'duplicate_product'
  | 'rename_spec_field' | 'soft_delete_spec_field' | 'delete_spec_field'
  | 'login' | 'logout' | 'session_revoked';

export async function logAudit(tx: Tx, args: {
  actorEmail: string;
  action: AuditAction;
  entityType: string;
  entityId: string;
  before: unknown;
  after: unknown;
  ip?: string;
  userAgent?: string;
}) {
  await tx.insert(auditLog).values({
    actorEmail: args.actorEmail,
    action: args.action,
    entityType: args.entityType,
    entityId: args.entityId,
    beforeJson: args.before as Record<string, unknown> | null,
    afterJson: args.after as Record<string, unknown> | null,
    ip: args.ip,
    userAgent: args.userAgent,

}
```

**Critical:** `tx` is the transaction object passed to `dbTx.transaction(async (tx) => { ... })` so the audit row commits/rolls back atomically with the mutation. Never call `logAudit` outside a transaction.

### `src/lib/csv.ts` (NEW — ADMIN-12)

**Analog:** `src/lib/slug.ts:1-41` (pure-string utility shape)

**Pattern (RESEARCH.md §Code Examples):**
```typescript
const NEEDS_QUOTE = /[",\n=]/;

function field(v: unknown): string {
  const s = v == null ? '' : String(v);
  if (NEEDS_QUOTE.test(s) || s.startsWith(' ') || s.endsWith(' ')) {
    return `"${s.replace(/"/g, '""')}"`;
  }
  return s;
}

export function toCsv<T extends Record<string, unknown>>(
  rows: T[],
  cols: (keyof T)[],
): string {
  const head = cols.map(field).join(',');
  const body = rows.map((r) => cols.map((c) => field(r[c])).join(',')).join('\r\n');
  return '﻿' + head + '\r\n' + body;  // UTF-8 BOM for Excel
}
```

import { dbTx } from '@/db/client-ws';                    // ← WS Pool client (transactions)
import { /* entity tables */ } from '@/db/schema';

export const save<Entity> = withAdminAction(saveSchema, async (input, ctx) => {
  // Read `before` snapshot (outside tx — small inefficiency, acceptable)

    ? await dbTx.select().from(entityTable).where(eq(entityTable.id, input.id)).limit(1)
    : null;

  const result = await dbTx.transaction(async (tx) => {
    // 1. base entity (insert OR update)
    const [row] = input.id
      ? await tx.update(entityTable).set({ /* fields */ })
          .where(eq(entityTable.id, input.id)).returning()
      : await tx.insert(entityTable).values({ /* fields */ }).returning();

    for (const locale of ['uz', 'ru', 'en'] as const) {
      const t = input.translations[locale];
      await tx.insert(entityTranslations)
        .values({ entityId: row.id, locale, ...t })
        .onConflictDoUpdate({
          target: [entityTranslations.entityId, entityTranslations.locale],
          set: t,
        });
    }

    // 3. per-entity extras (e.g., spec values for products: DELETE + INSERT)

    // 4. audit row INSIDE the same tx
    await logAudit(tx, {
      actorEmail: ctx.actorEmail,
      action: input.id ? 'update' : 'create',
      entityType: '<entity>',
      entityId: row.id,
      before: before?.[0] ?? null,
      after: row,
      ip: ctx.ip,
      userAgent: ctx.userAgent,
    });

    return row;
  });

  return result;

## Anti-Patterns (call out in plans, lint where possible)

From RESEARCH.md §Anti-Patterns to Avoid:

1. **`db.transaction(...)` with HTTP client** → runtime "No transactions support in neon-http driver." Always `dbTx`. (PITFALL #1)
2. **`revalidateTag` inside `dbTx.transaction(...)`** → cache invalidated on rollback. Always after commit. (PITFALL #2)
3. **Per-field fallback in admin display** → admin queries return raw `null`; fallback only in Phase 3 public reads. (PITFALL #6)
4. **Hand-rolling JSON CRUD endpoints under `/api/`** → Server Actions only. (`/api/` reserved for Cloudinary sign + Auth.js callbacks)
5. **Custom upload route proxying file bytes** → Cloudinary direct upload only.
6. **Hard-deleting `spec_field` without confirmation modal** → D-07 mandates confirm dialog with affected count. (PITFALL #14)
7. **Storing Auth.js session token client-side** → already mitigated; do not augment.
8. **`drizzle-kit db:push` against production** → CLAUDE.md forbids. Generate → review → migrate.
9. **`revalidateTag(tag)` single-arg** → TS error in Next 16. Always `revalidateTag(tag, 'max')`.
10. **TanStack client-side sort over server-paginated rows** → set `manualPagination: true, manualSorting: true, manualFiltering: true`. (PITFALL #8)
