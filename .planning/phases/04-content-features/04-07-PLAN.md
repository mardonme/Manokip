<objective>
This plan ships the recipe admin UI surface — the marquee deliverable for CONT-01. Three new client components (RecipeBodyEditor, LinkedProductsPicker, RecipeForm wrapping both) + four new admin route files (list page + new + edit + the table client island) + sidebar nav update.

The Tiptap admin editor mounts `useEditor` with `immediatelyRender: false` per P4-1 (CRITICAL — Tiptap v3 + Next 16 RSC compatibility; without this flag the page hydration mismatches on every render). Image insertion goes through the existing `<CldUploadWidget>` with `options.folder='recipes'` — the FOLDER_ALLOWLIST extension landed in plan 04-02 makes this work end-to-end without a new server endpoint. The custom CloudinaryImage extension stores `attrs.publicId` (NOT a Cloudinary URL) so the body JSON is account-portable.

LinkedProductsPicker uses shadcn Combobox (Popover + Command) with client-side text filter — RESEARCH §Linked-products picker confirms client-side is sufficient at v1 scale (≤200 published products). Pre-fetched in the parent RSC; client island manages the value array via RHF useFieldArray + dnd-kit drag-reorder.

Closure status: Standard execute plan with 1 client-side TDD checkpoint (jsdom specs flipped GREEN). Exit gate: tsc clean + 4+ jsdom specs green + admin recipes routes accessible end-to-end.
</objective>

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin browser → Cloudinary direct (signed widget) | bytes never round-trip through Vercel; signature minted by /api/cloudinary/sign with FOLDER_ALLOWLIST gate (extended in 04-02 with 'recipes') |
| Admin form submit → saveRecipe / lifecycle actions | RHF + Zod validate client-side; Server Action re-validates with recipeInsertSchema.parse() (defense-in-depth) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| P4-1 | Tampering | RSC hydration mismatch on Tiptap admin editor | mitigate | useEditor passed `immediatelyRender: false`; spec asserts the option. Without this flag, every page render mismatches hydration → React errors in production. |
| T-04-XSS-04 | XSS | Tiptap image attrs.src as data: URI bypassing Cloudinary | mitigate | CloudinaryImage extension stores publicId attribute; admin UI inserts only via CldUploadWidget which returns Cloudinary public_id; src is never read by the public renderer (nodeMapping override). |
| T-04-AUTH-01 | Spoofing | Direct call to saveRecipe / lifecycle actions bypassing form | mitigate | All actions wrapped by withAdminAction (admin session + audit) — bypassing the form just means submitting JSON directly to the action; the auth + audit gate is unchanged. |
| T-04-DOS-01 | DoS | Tiptap doc with thousands of nodes | accept | v1 trust posture: only invited admins author. Re-evaluate Phase 5 with a `JSON.stringify(body).length` cap at saveRecipe-tx entry if abuse appears. |

</threat_model>
