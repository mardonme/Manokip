<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin form submit → saveRecipe Server Action | input parsed by recipeInsertSchema (Zod refinement enforces reserved-slug denylist + locale enum + uuid format); withAdminAction enforces admin session via requireAdmin() |
| saveRecipe → dbTx.transaction | atomic 5-step write; FK cascade drops dependents on entity delete |
| Post-tx revalidate fan-out | NEVER inside the tx lambda (Pitfall #2); union of old+new linkedProductIds drives the fan-out |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-AUTH-01 | Spoofing | Unauthenticated saveRecipe / publish / unpublish / delete call | mitigate | All 4 actions wrapped by withAdminAction (admin session required); structurally enforced by Phase 2 plan 02-04 wrapper. |
| T-04-NREP-01 | Repudiation | Admin denies publishing a recipe with defamatory content | mitigate | audit_log writes before+after on every publishRecipe / unpublishRecipe / saveRecipe / deleteRecipe call. before_json captures Tiptap doc on update; after_json=NULL on delete (per audit.ts contract). |
| T-04-INFO-01 | Information Disclosure | saveRecipe leaking draft content via revalidateUsedIn cascading to public pages | mitigate | revalidateUsedIn invalidates cache only — content visibility is enforced at READ time by getUsedInForProduct querying the pgView (which filters WHERE status='published') and getRecipeBySlug querying recipes WHERE status='published'. Cache invalidation is orthogonal to access control. |
| P4-5 | Tampering | Forgetting to enumerate old linked productIds before tx commit (used-in cache stale) | mitigate | saveRecipe + deleteRecipe both pre-tx-capture oldLinkedProductIds. publishRecipe + unpublishRecipe read currently-linked productIds from product_recipes WHERE recipeId=id. Spec 7 asserts the union fan-out for saveRecipe replace-on-save. |

</threat_model>
