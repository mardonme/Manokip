This plan installs the Tiptap stack and wires up the shared extension array + server-side renderer. It is upstream of every code-producing plan in Phase 4 — the admin editor (04-07/04-08), public detail pages (04-09/04-10), and lib helpers (04-03) all import from `src/lib/tiptap-extensions.ts` or `src/lib/tiptap-render.ts`. The Cloudinary signing endpoint allow-list extension is the one-line edit that lets Tiptap inline image uploads reuse the Phase 2 widget paramsToSign protocol verbatim — no new endpoint, no new client code path. <threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public RSC → Tiptap static-renderer | renderer receives jsonb body authored by admin (untrusted-by-default text content); HTML output emitted to public visitor browser |
| Admin client → /api/cloudinary/sign (widget paramsToSign protocol) | widget POSTs `{ paramsToSign: { folder: 'recipes' \| 'industries', timestamp, ... } }`; endpoint validates folder against allow-list before signing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-XSS-01 | XSS | Tiptap body rendered to HTML on public page | mitigate | Static-renderer locked extension allow-list (TIPTAP_EXTENSIONS array IS the allow-list) + built-in `escapeHTML` + `escapeHTMLAttribute` for text + attrs (verified at dist:line in RESEARCH); `unhandledNode: () =&gt; ''` softens unknown-node throw to silent drop. Spec 1 in tests/lib/tiptap-render.test.ts is the regression-locker. |
| T-04-XSS-04 | XSS | Tiptap image extension with `data:` URI src | mitigate | CloudinaryImage extension stores `publicId` (not URL); public RSC nodeMapping reads ONLY `attrs.publicId`. `data:` URIs in `src` are dropped because the override doesn't pass src through. |
| T-04-SSRF-01 | SSRF | Tiptap image upload signing endpoint accepting attacker-controlled folder | mitigate | FOLDER_ALLOWLIST gate in /api/cloudinary/sign extended with 'recipes' + 'industries'. Closed enum; no folder outside the 4-element list passes validation. T-CLD-02 from Phase 2 preserved. |

</threat_model>
