<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin form submit → saveIndustry Server Action | input parsed by industryInsertSchema (Zod refinement enforces reserved-slug denylist + locale enum + uuid format); withAdminAction enforces admin session |
| saveIndustry → dbTx.transaction | atomic 5-step write; FK cascade drops dependents on entity delete |
| Post-tx revalidate fan-out | NEVER inside tx lambda; union of old+new linkedProductIds drives the fan-out |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-AUTH-01 | Spoofing | Unauthenticated saveIndustry / publish / unpublish / delete call | mitigate | All 4 actions wrapped by withAdminAction; structurally enforced. |
| T-04-NREP-01 | Repudiation | Admin denies publishing an industry with sensitive content | mitigate | audit_log writes before+after on every mutation; before_json captures Tiptap doc on update; after_json=NULL on delete. |
| T-04-INFO-01 | Information Disclosure | Draft industry leaking via revalidate fan-out | mitigate | Revalidate is cache-invalidation only; visibility enforced at READ time (getIndustryBySlug + product_used_in_v both filter WHERE status='published'). |
| P4-5 | Tampering | Forgetting to enumerate old linked productIds before tx commit | mitigate | saveIndustry + deleteIndustry both pre-tx-capture oldLinkedProductIds. publishIndustry + unpublishIndustry read currently-linked productIds. Spec 7 asserts the union fan-out. |

</threat_model>
